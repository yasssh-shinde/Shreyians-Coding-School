Part 1: Basics of JavaScript

1.1 Variables & Hoisting

Variables are containers for storing data values. JavaScript has three ways to declare variables: var, let, and const.

Understanding Variable Hoisting

Hoisting is JavaScript's behavior of moving declarations to the top of their scope before code execution. This is important to understand:

üìå Key Concept: Variable and function declarations are "hoisted" (moved to the top), but not their assignments/initializations.
üìå Example: var Hoisting
JavaScript
// This works! (var is hoisted) console.log(x); // undefined var x = 5; console.log(x); // 5

What happens internally:

Actual Execution
var x; // Hoisted and initialized as undefined console.log(x); // undefined x = 5; console.log(x); // 5

var vs let vs const

Feature var let const
Scope Function-scoped Block-scoped Block-scoped
Re-declare ‚úÖ Yes ‚ùå No ‚ùå No
Update ‚úÖ Yes ‚úÖ Yes ‚ùå No
Hoisting Initialize as undefined Temporal Dead Zone Temporal Dead Zone

Part 2: Functions in JavaScript

2.1 Understanding Functions

Functions are reusable blocks of code designed to perform a particular task. They accept input (parameters), process it, and optionally return output.

Function Parameters & Arguments

Parameters: Variables in function definition

Arguments: Actual values passed when calling the function

Function Types

üìå Example: Function Declaration vs Expression
JavaScript
// Function Declaration (Hoisted) function add(a, b) { return a + b; } // Function Expression (Not Hoisted) const subtract = function(a, b) { return a - b; }; console.log(add(5, 3)); // 8 console.log(subtract(5, 3)); // 2

Arrow Functions

Modern syntax introduced in ES6. Has lexical this binding (doesn't have its own this).

üìå Example: Arrow Functions
JavaScript
// Basic arrow function const multiply = (a, b) => a * b; console.log(multiply(5, 3)); // 15 // Single parameter (parentheses optional) const square = x => x * x; console.log(square(4)); // 16

3.2 Objects in JavaScript

Objects are key-value pairs used to store related data and functionality.

Object Methods

üìå Example: Object Methods
JavaScript
const person = { name: "Alice", age: 25 }; // Object.keys() - get all keys console.log(Object.keys(person)); // ["name", "age"] // Object.values() - get all values console.log(Object.values(person)); // ["Alice", 25] // Object.entries() - get [key, value] pairs console.log(Object.entries(person)); // [["name", "Alice"], ["age", 25]] // Object.freeze() - prevent all changes Object.freeze(person); // Object.seal() - prevent add/delete Object.seal(person);

Cloning Objects (Shallow & Deep)

üìå Example: Object Cloning
JavaScript
const original = { name: "Alice", skills: ["JS", "React"] }; // Shallow clone (spread operator) const shallowClone = { ...original }; shallowClone.name = "Bob"; // Doesn't affect original shallowClone.skills.push("Node"); // DOES affect original! // Deep clone (JSON method) const deepClone = JSON.parse(JSON.stringify(original)); deepClone.skills.push("Python"); // Doesn't affect original

Part 6: Browser APIs & Storage

6.1 Browser Object Model (BOM)

The Browser Object Model provides objects for interacting with the browser.

Window Object

üìå Example: Window Object
JavaScript
// Window methods window.alert("Alert Message"); window.confirm("Confirm?"); // true/false window.prompt("Enter name:", "default"); // Returns input // Window properties console.log(window.innerWidth); // Viewport width console.log(window.innerHeight); // Viewport height console.log(window.location); // Current URL console.log(window.history); // Browsing history

6.2 Local Storage & Session Storage

LocalStorage: Persists indefinitely (no expiry)

SessionStorage: Cleared when browser tab closes

üìå Example: LocalStorage
JavaScript
// Store data localStorage.setItem("username", "Alice"); // Retrieve data const username = localStorage.getItem("username"); // Store objects const user = { name: "Alice", age: 25 }; localStorage.setItem("user", JSON.stringify(user)); // Retrieve objects const retrievedUser = JSON.parse(localStorage.getItem("user")); // Remove specific item localStorage.removeItem("username"); // Clear all localStorage.clear();

6.3 Timing Events (setTimeout & setInterval)

Schedule code execution after a delay or at regular intervals.

üìå Example: setTimeout
JavaScript
// Execute after 2 seconds const timeoutId = setTimeout(() => { console.log("Executed after 2 seconds"); }, 2000); // Cancel timeout clearTimeout(timeoutId);
üìå Example: setInterval
JavaScript
// Execute every 1 second let count = 0; const intervalId = setInterval(() => { count++; console.log("Count:", count); // Stop after 5 times if (count === 5) { clearInterval(intervalId); } }, 1000);
